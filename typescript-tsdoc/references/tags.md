# TSDoc — разрешённые теги

Опорный материал: [tsdoc.org/pages/spec/tag_kinds/](https://tsdoc.org/pages/spec/tag_kinds/).

TSDoc различает три вида тегов: **block**, **modifier**, **inline**.

## Block-теги

Блочные теги располагаются в теле комментария после summary, каждый с новой строки.

| Тег               | Назначение                                                                                | Пример                                            |
|-------------------|-------------------------------------------------------------------------------------------|---------------------------------------------------|
| `@remarks`        | Расширенное описание: инварианты, поведение, ограничения, которые не умещаются в summary. | `@remarks\nБросает при пустом вводе.`             |
| `@privateRemarks` | Внутренние заметки для мейнтейнеров, не попадают в публичную документацию.                | `@privateRemarks\nПоправить после миграции.`      |
| `@param`          | Описание параметра. По одному тегу на параметр, в порядке сигнатуры.                      | `@param id — Идентификатор пользователя.`         |
| `@typeParam`      | Описание параметра-типа дженерика.                                                        | `@typeParam T — Тип элемента коллекции.`          |
| `@returns`        | Что возвращает функция. Не дублировать тип.                                               | `@returns Массив активных пользователей.`         |
| `@throws`         | Тип исключения и условие, при котором оно бросается. Один тег на один тип.                | `@throws {ValidationError} Если email невалиден.` |
| `@defaultValue`   | Значение по умолчанию для опционального поля/параметра интерфейса.                        | `@defaultValue `'default'`.`                      |
| `@deprecated`     | Пометка устаревшим. Всегда указывать замену.                                              | `@deprecated Используйте {@link newFn}.`          |
| `@see`            | Ссылка на связанный символ или документ.                                                  | `@see {@link OtherClass}`                         |
| `@example`        | Пример использования. **Добавлять только по явной просьбе пользователя.**                 | `@example\n\`\`\`ts\nfoo(1);\n\`\`\``             |
| `@decorator`      | Используется в объявлениях декораторов.                                                   | `@decorator `@Injectable()`.`                     |

## Modifier-теги

Модификаторы помечают сам символ — ставятся отдельной строкой без параметров. Обычно идут в
конце блока.

| Тег                     | Назначение                                                                   |
|-------------------------|------------------------------------------------------------------------------|
| `@public`               | Явно публичный символ (по умолчанию всё экспортированное публично).          |
| `@alpha`                | Нестабильное API, экспериментальное.                                         |
| `@beta`                 | API в бете, может измениться.                                                |
| `@experimental`         | Синоним `@alpha`/`@beta` в некоторых тулах.                                  |
| `@internal`             | Не часть публичного API, но экспортировано технически.                       |
| `@override`             | Метод переопределяет родительский.                                           |
| `@sealed`               | Класс не предназначен для наследования.                                      |
| `@virtual`              | Метод спроектирован для переопределения.                                     |
| `@readonly`             | Свойство доступно только для чтения (когда TS-модификатор не может сказать). |
| `@eventProperty`        | Свойство является событием (emitter-стиль).                                  |
| `@packageDocumentation` | Блок — документация всего пакета. Пишется в `index.ts` корня.                |

## Inline-теги

Inline-теги вкладываются в описание через `{@tag ...}`.

| Тег                    | Назначение                                                    | Пример                           |
|------------------------|---------------------------------------------------------------|----------------------------------|
| `{@link target}`       | Ссылка на другой символ. Можно с текстом: `{@link xподпись}`. | `См. {@link User.id}.`           |
| `{@inheritDoc target}` | Унаследовать описание от другого символа.                     | `{@inheritDoc BaseClass.method}` |
| `{@label NAME}`        | Пометить перегрузку/сигнатуру для последующих ссылок.         | `{@label STRING_OVERLOAD}`       |

## Запрещено

Эти теги **не использовать** — они вне TSDoc-спецификации (это JSDoc/классический JSDoc):

- `@type`, `@function`, `@class`, `@method`, `@constructor`, `@interface`
- `@kind`, `@module`, `@namespace`, `@memberof`
- `@property`, `@prop`
- `@arg`, `@argument` (использовать `@param`)
- `@return` в единственном числе (только `@returns`)
- `@exception` (использовать `@throws`)
- `@author`, `@version`, `@since` (хранить в CHANGELOG/git)
- `@todo`, `@fixme` (использовать `// TODO:` в теле кода)
